# Ma'lumotnoma tizimi — Implementation Roadmap

Spec: [`../specs/2026-07-15-malumotnoma-system-design.md`](../specs/2026-07-15-malumotnoma-system-design.md)

The build is split into sequential plans. Each produces working, testable software and is written
in full detail **just before** it executes (so it stays accurate against the evolving code).

| # | Plan | Deliverable | Status |
|---|------|-------------|--------|
| 1 | **Foundation** | npm-workspace monorepo; `@spravka/shared` (`/db` schema+client, `/core` auth+workflow+numbering); qrcode-pro absorbed as `apps/web-qr` on the shared DB; seed. Monorepo builds, web-qr runs as before, core logic unit-tested. | **written** |
| 2 | **Shared UI** | `@spravka/shared/ui` — AppShell/sidebar, DataTable, LoginPage, Modal, forms, Toast, timeline, calendar & monitoring widgets, theme/i18n/icons (ported from credit-core, trimmed). Storybook-less: a demo route renders each. | pending |
| 3 | **web-yurist** | Lawyer app: login, dashboard, create/edit/submit ariza (firm picker + fiz-litso + contract fields), own list, Kalendar, profil. Workflow SUBMIT wired. | pending |
| 4 | **web-admin** | Admin app: Monitoring dashboard, Arizalar inbox (view/edit-until-approval/approve/return), Firms CRUD, Users CRUD, Kalendar. Workflow APPROVE/RETURN wired. Edit-lock enforced. | pending |
| 5 | **web-rahbar + PDF + public + devops** | Director app: sign (→ PDF render with «ТАСДИҚЛАНДИ» stamp), delete/restore arxiv, monitoring. web-qr public `/m/[id]` (serve PDF / TASDIQLANMAGAN + print). One docker-compose + nginx subdomains; cutover + QR-data migration. | pending |

**Cross-cutting conventions (all plans):**
- Enums: const-object + union pattern (Prisma-assignable), mirrored in `@spravka/shared/core`.
- Tests: **Vitest** for `@spravka/shared/core` pure logic; Next.js route handlers tested via direct function calls where practical.
- Commits: frequent, one per task step group. Conventional-commit prefixes.
- Uzbek Latin UI, Cyrillic document body.
